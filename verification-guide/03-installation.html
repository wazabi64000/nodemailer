Installation du projet

Voici les étapes pour initialiser un projet de vérification de compte avec Express et MongoDB :

1. Créer la structure du projet

mkdir verify-app && cd verify-app
mkdir controllers models routes config middlewares utils
touch .env index.js README.md
touch controllers/userController.js models/User.js routes/userRoutes.js config/db.js middlewares/errorHandler.js utils/mailer.js

2. Initialiser npm et installer les dépendances

npm init -y
npm install express mongoose dotenv jsonwebtoken nodemailer cors

3. Installer les outils de développement

npm install --save-dev nodemon

Ajoute ce script dans ton package.json :

"scripts": {
  "dev": "nodemon index.js"
}

Structure du projet et rôle des fichiers

  • controllers/ — Contient la logique métier : ce que fait chaque route. Exemple : vérifier un token, envoyer un email.
  • models/ — Définit les schémas Mongoose (structure de la base de données MongoDB).
  • routes/ — Regroupe toutes les routes de l’application (API REST, ex: /api/users).
  • config/ — Contient la configuration comme la connexion à MongoDB.
  • middlewares/ — Fonctions intermédiaires entre la requête et la réponse. Exemple : gestion des erreurs.
  • utils/ — Fonctions utilitaires comme l’envoi d’emails avec Nodemailer.
  • .env — Contient les variables sensibles (clés API, mots de passe, URLs de base de données).
  • index.js — Point d’entrée du serveur Express.
  • README.md — Documentation du projet.
Conseil : Structurer ton projet dès le départ améliore la lisibilité pour tous, y compris les personnes utilisant un lecteur d'écran ou une interface en mode texte.

4. Lancer le projet

npm run dev

Le serveur démarre sur localhost:5000 (ou selon ta configuration).

Astuce : Utilise une structure claire dès le début pour t’y retrouver facilement, surtout si tu travailles en équipe ou si tu es malvoyant (les lecteurs d’écran s’adaptent mieux à une arborescence logique).
